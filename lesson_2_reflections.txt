- What happens when you initialize a repository? Why do you need to do it?

A git folder gets created in the directory. You need to do it in order to use
git with the files you have.

- How is the staging area different from the working directory and the
repository? What value do you think it offers?

The staging area is an intermediate place between the directory and the 
repository. It offers flexibility over which files to commit at once.

- How can you use the staging area to make sure you have one commit per logical
change?

You can update the staging area many times and use the diff command to see if
the change is enough to commit.

- What are some situations when branches would be helpful in keeping your 
history organized? How would branches help?

It would be helpful when working with alternative layouts or alternative
feature sets. You could go on editing without breaking the working version.
